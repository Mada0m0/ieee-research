This project is an automated control algorithm research and code generation platform that uses **Jules AI** to directly convert mathematical models extracted from academic papers into production-level Python code, and automatically creates PRs and runs CI/CD for quality verification.

## Architecture Overview

```
Research layer (hermes) Translation layer (Jules API) Code layer (GitHub)
ieee_search.py вЂ‹вЂ‹в”Ђв”Ђв†’ paper_analyzer.py в”Ђв”Ђв†’ Jules job JSON в”Ђв”Ђв†’ Generate code + PR в”Ђв”Ђв†’ Merge + CI/CD
(Extract mathematical model) (including prompt + source)
```

Core components:
- **`hub.py`** вЂ“ Central dispatch script (search, analyze, trigger Jules)
- **`agents/hermes/`** вЂ“ paper search and in-depth analysis tool (ieee-search skill)
- **`jules_jobs/`** вЂ“ Pre-built Jules job JSON files
- **`implementations/`** вЂ“ final code generated by Jules (organized by categories)
- **`.github/workflows/pipeline.yml`** вЂ“ CI/CD pipeline (code style, type checking, unit testing)

## Researchв†’Code Pipeline

The complete process is divided into 6 steps, which can be executed automatically through `hub.py pipeline` or manually followed step by step.

### 1. Search papers

```bash
python agents/hermes/ieee_search.py --query "Bouc-Wen piezoelectric hysteresis" --max 5
```

Output JSON results (including DOI, title, abstract, citation count).

### 2. In-depth analysis

```bash
python agents/hermes/paper_analyzer.py --doi "10.1109/ACCESS.2020.2984645" --lang zh --output research/reports/analysis.md
```

Generate Markdown report, including:
- Original abstract, methodological classification, and innovation points
- Relevance rating to user research area (1вЂ“5 stars)
- Technical points that can be learned from

### 3. Build Jules Job JSON

Extracted from the analysis report:
- **Prompt**: includes paper background, mathematical model, engineering requirements (file path, class name, method signature, dependency constraints, test requirements)
- **sourceContext**: Specify the GitHub repository and starting branch
- **automationMode** = `"AUTO_CREATE_PR"`

Example structure (refer to `jules_jobs/rmso_bw_fuzzy_nn.json`):

```json
{
  "prompt": "## Task: Implement RMSO-BW Hysteresis Model...\n...",
  "sourceContext": {
    "source": "sources/github/Mada0m0/ieee-research",
    "githubRepoContext": { "startingBranch": "main" }
  },
  "title": "RMSO-BW Hysteresis Model with Fuzzy-NN Controller",
  "automationMode": "AUTO_CREATE_PR"
}
```

**вљ пёЏ IMPORTANT**: `sourceContext` must contain `githubRepoContext.startingBranch`, otherwise the API returns a 400 error (old format is obsolete).

### 4. Upload the reference document and create a branch

Put the job files into `jules_jobs/` and the paper references into `references/`, then push to a new branch of the remote repository:

```bash
git checkout -b feat/new-model
git add references/paper.md jules_jobs/new_model.json
git commit -m "feat: add Jules job for XXX model"
git push origin feat/new-model
```

**Strong blocking verification**: Wait for the remote branch to be fully available (use the `wait_for_remote_branch` function in `hub.py` to avoid Jules cloning failure due to the branch not being ready).

### 5. Trigger Jules API

```bash
curl -X POST https://jules.googleapis.com/v1alpha/sessions \
  -H "Content-Type: application/json" \
  -H "X-Goog-Api-Key: $JULES_API_KEY" \
  -d @jules_jobs/new_model.json
```

Returns the `session` ID and web URL. Jules will:
- Create a new working branch from the `main` branch (name format: `feat/model-name-<session_id>`)
- Generate all files (code, tests, examples)
- Submit and create Pull Request

### 6. Merge PR and run CI/CD

After a PR is created, GitHub Actions automatically execute:
- `pycodestyle` code style check
- `mypy` type annotation verification
- `pytest` unit tests (must all pass)
- If passed, it is safe to merge.

After merging, the generated code is included in the `implementations/` master branch.

## How to add a new paper

1. **Search and Analysis**:
   ```bash
   python agents/hermes/ieee_search.py --doi "10.1109/XXX.2024.1234567" --max 1
   python agents/hermes/paper_analyzer.py --doi "10.1109/XXX.2024.1234567" --lang zh
   ```

2. **Create reference document**: Write the abstract, methodology, and key formulas into `references/paper_author_year.md`.

3. **Write Jules Job**: Use the template `jules_jobs/template.json` and fill in the prompt (refer to the existing job).

4. **Local test** (optional): You can first verify whether the prompt is clear in the simulation environment.

5. **Trigger pipeline**:
   ```bash
   git checkout -b feat/new-paper-model
   git add references/... jules_jobs/...
   git commit -m "feat: add Jules job for XXX"
   git push origin feat/new-paper-model
# Wait for the branch to be visible before triggering the API
   curl -X POST ... -d @jules_jobs/new_model.json
   ```

6. **Review PR**: The PR generated by Jules will be automatically assigned to you, and merged after checking the code quality.

## Jules API Integration Notes

- **Authentication**: Use `X-Goog-Api-Key` header with value format `AQ.xxxx...` (Google Cloud API style).
- **Endpoint**: `https://jules.googleapis.com/v1alpha/sessions`
- **source name**: must be exactly what is returned by the `listSources` API, for example `sources/github/Mada0m0/ieee-research`.
- **Source Context**: `githubRepoContext.startingBranch` must be provided, otherwise the request will be rejected.
- **Job File Encoding**: Newlines in prompts must be escaped with `\n` (or handled automatically using `json.dump`) to avoid JSON parsing errors.
- **Branch Visibility**: You need to wait 2вЂ“3 seconds after pushing before triggering Jules, otherwise it may cause `fatal: reference is not a tree`.

## CI/CD pipeline description

File: `.github/workflows/pipeline.yml`

Trigger conditions:
- push to `main`
- Any PR created/updated

Execution steps:
1. **Code style**: `pycodestyle implementations/tests/examples/`
2. **Type check**: `mypy --ignore-missing-imports implementations/`
3. **Unit tests**: `pytest tests/ -v --cov=implementations`
4. **Build Report**: Generate test coverage and style reports (`main` branch only).

All stages must pass before the PR can be merged.

## Contribution Guidelines

- New algorithm implementations should be automatically generated via Jules to avoid manually submitting code to `implementations/`.
- If you find that the generated code is defective, please modify the prompt in the corresponding Jules Job JSON and retrigger the task (new branch).
- Update research domain configuration: edit `research/templates/research_domains.yaml`.
